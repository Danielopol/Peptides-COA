Three palette laws (carry over the existing comments in verdict.dart — they're correct): ① verdict colors color the signal only (tag, icon, arc, purity chip) — never card backgrounds or chrome; ② completeness stays teal — an inventory, not a judgment; ③ the lime XP color is for effort (streaks, ranks, badges) and may never appear on anything describing the document. 2 · ThemeData — drop-in for core/theme.dart

class HelixTheme {
  const HelixTheme._();

  static ThemeData dark() {
    const c = HelixColors.dark; // token class from §1
    final scheme = const ColorScheme.dark().copyWith(
      surface: c.bg,                 // scaffold uses bg
      surfaceContainer: c.surface,   // cards
      surfaceContainerHigh: c.surface2,
      surfaceContainerHighest: c.surface3,
      primary: c.accent, onPrimary: const Color(0xFF04131E),
      secondary: c.cTeal,
      onSurface: c.ink, onSurfaceVariant: c.ink2,
      outline: c.line, outlineVariant: c.line2,
      error: c.vRed,
    );
    final base = ThemeData(useMaterial3: true, colorScheme: scheme,
        scaffoldBackgroundColor: c.bg);
    return base.copyWith(
      textTheme: _textTheme(base.textTheme, c),
      appBarTheme: AppBarTheme(centerTitle: false, elevation: 0,
          scrolledUnderElevation: 0, backgroundColor: c.bg,
          foregroundColor: c.ink),
      cardTheme: CardThemeData(elevation: 0, clipBehavior: Clip.antiAlias,
          color: c.surface,
          shape: RoundedRectangleBorder(
            borderRadius: BorderRadius.circular(14),          // was 16
            side: BorderSide(color: c.line))),
      filledButtonTheme: FilledButtonThemeData(style: FilledButton.styleFrom(
          minimumSize: const Size(0, 52),
          backgroundColor: c.accent, foregroundColor: const Color(0xFF04131E),
          textStyle: GoogleFonts.archivo(fontSize: 15.5, fontWeight: FontWeight.w600),
          shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(10)),
          // neon halo lives on a wrapper, not the button:
          // DecoratedBox(boxShadow: [BoxShadow(color: c.accentGlow,
          //   blurRadius: 22, spreadRadius: -4)])
      )),
      outlinedButtonTheme: OutlinedButtonThemeData(style: OutlinedButton.styleFrom(
          minimumSize: const Size(0, 52), foregroundColor: c.ink,
          side: BorderSide(color: c.line))),
      dividerTheme: DividerThemeData(color: c.line2, thickness: 1, space: 1),
      chipTheme: ChipThemeData(  // specimen-tag chips: squarish, mono
          backgroundColor: c.surface3, side: BorderSide(color: c.line),
          shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(6)),
          labelStyle: GoogleFonts.ibmPlexMono(fontSize: 11,
              fontWeight: FontWeight.w500, color: c.ink2)),
    );
  }
}

4 · Motion spec — the scan-reveal ritual

Total budget ≈3s from response-received to settled screen. Every step respects MediaQuery.disableAnimations — reduced motion shows final states with no sweeps and no confetti.

StepTiming / curveFlutter notes
1 · beam sweep2.4s/loop, ease-in-out, repeats while waiting2px accent line over skeleton doc, BoxShadow(accentGlow, blur 18, spread 4) + 38px gradient trail. Rule counter ticks 0→82 (~90ms/increment, random 1–4 steps). Status copy crossfades every ~950ms: READING DOCUMENT TEXT… → MATCHING LAB TEMPLATES… → CROSS-CHECKING MOLECULAR WEIGHTS… → HUNTING FORGERY MARKERS…
2 · ticks in240ms total, 12ms staggerGauge calibration ticks fade in before the arc — the instrument "powers on".
3 · arc + count1100ms, Cubic(0.16, 1, 0.3, 1)Single controller drives arc sweep 0→score and numeral count-up ((score×eased).round()). Glow rides the arc tip.
4 · verdict tag260ms, Cubic(0.2, 1.3, 0.4, 1)Scale 1.06→1.0 + fade — a stamp settling. One light haptic (HapticFeedback.lightImpact) exactly here, nowhere else.
5 · confettistarts with step 4, ≤2.5s, onceAuthentic only. Skip entirely when ludic flag = clinical.
6 · cascade320ms each, 60–80ms stagger, ease-outCards below hero: 12px slide-up + fade, top-to-bottom. Critical (red) finding tiles arrive last and pulse their border once — no shake, never alarmist.

Micro-interactions

  • buttons — pressed: scale 0.98, 120ms + surface3 wash. Cards never lift on dark; border brightens to ink3 instead.
  • checklist rows — ✓ ticks on scroll-into-view, 26ms stagger ("inventory being counted").
  • XP line — fades in 120ms after verdict copy. Values: clean scan +40 · caution +60 · red flag caught +120 (catching problems must pay best).
  • badge unlock — 520ms Cubic(0.2, 1.4, 0.4, 1), translateY 18→0 + scale 0.85→1.04→1.0.

5 · Guardrails (product-level, enforce in review)

  • No badge or XP for "authentic" outcomes as such — rewards attach to verification habits (scans done, source checks, batch matches, fine print read). Lime never touches document semantics.
  • LimitationsCard is structural, not dismissible, and ships in every results state including errors-adjacent ones.
  • Verbatim strings — verdict copy, finding messages, limitation clauses and the disclaimer come from the backend contract untouched; design only stages them.
  • Glow is earned — only the verdict arc/numeral, primary CTA halo, and XP bar may emit light. Adding glow elsewhere requires removing it somewhere else.
  • Light mode exists (§1 tokens) but loses all glow, lattice, and confetti — it's the clinical-paper rendering of the same hierarchy.
