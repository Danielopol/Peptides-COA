FORENSIC RULESET SUMMARY (apply all applicable rules):

STRUCTURE CHECKS (verify presence of each):
- Product name matches stated compound
- Unique batch/lot number present and non-generic
- Specific analysis date present (not future-dated)
- Named third-party testing laboratory
- Laboratory address and contact information
- Analyst signature or certification
- Numerical purity percentage (not qualitative)
- Mass spectrometry identity data with observed molecular weight

NUMERICAL CHECKS (verify values):
- HPLC purity ≥95% (standard RUO); ≥98% (advanced); ≥99% (pharmaceutical)
- Purity has specific non-round decimal (not exactly N.00%)
- Purity does not exceed 100%
- MS mass error within tolerance (MALDI-TOF ±0.1–0.3 Da; ESI-MS ±0.05 Da; general ±1 Da)
- Net Peptide Content (if reported): 60–90% range (never >95%)
- COA analysis date within 18 months of today (ideally within 90 days)
- HPLC detection wavelength at 214 nm or 220 nm
- Trace impurity peaks present in chromatogram for <99.9% purity claims

ANALYTICAL METHOD CHECKS:
- HPLC explicitly named as purity method
- MS explicitly named as identity method
- C18 column specified (or justified alternative)
- MS technique type specified (MALDI-TOF, ESI-MS, etc.)
- Chromatogram or peak table data attached

LABORATORY CREDENTIAL CHECKS:
- Lab is third-party independent (not vendor's in-house lab)
- ISO/IEC 17025 accreditation stated; accreditation body is a recognized ILAC member
- Lab independently discoverable via web search
- Digital verification key or QR code valid (if present)
- Lab is a known legitimate peptide testing laboratory (Janoshik, Colmaric, Vanguard, MZ Biolabs, Freedom Analytics, Novascreen, or equivalent verifiable lab)

FORMATTING CHECKS:
- Document resolution clear and professional throughout
- Uniform resolution across all sections (no localized pixelation in data fields)
- Consistent fonts in data fields and template
- No clone-stamp or copy-paste artifacts visible
- Document downloadable and fully readable

METADATA CHECKS:
- PDF creation date ≥ analysis date
- Authoring software is professional tool (not Photoshop, GIMP, or image editors)
- No future-dated analysis (any future date = AUTO FAIL)
- COA is batch-specific, not a generic template reused across products

CROSS-REFERENCE CHECKS:
- Batch number on COA matches batch number on product vial
- Observed MS mass matches independently calculated theoretical MW for named peptide
- Vendor name on COA matches purchasing vendor
- All dates are chronologically plausible (receipt ≤ analysis ≤ report ≤ today)
- If QR/verification key present: online data exactly matches paper COA

FORGERY INDICATOR CHECKS:
- No suspiciously round purity (not exactly 99.00% or 100.00%)
- No zero impurity peaks when purity claimed <100%
- No exact zero MS mass error (all instruments have non-zero measurement error)
- COA is not identical to COAs for other products from same vendor
- No identical HPLC retention times across different peptide products
- Official stamps/seals not blurry while surrounding text is sharp
- No localized pixelation in specific data fields (AUTO FAIL if present)
- Lab has record of vendor and batch when contacted (AUTO FAIL if denied)
- COA available pre-purchase (not withheld until after payment)
- Vendor does not discourage direct lab contact

AUTO-FAIL CONDITIONS (any one of these = score of 0, label REJECT):
1. Analysis date is in the future
2. Localized pixelation in specific data fields (value was digitally altered)
3. Lab denies issuing the COA when contacted
4. Digital verification key/QR code returns data inconsistent with paper COA
5. COA structure is identical across multiple different peptide products

SCORING:
- Assign each applicable rule: PASS or FAIL
- Rules that genuinely cannot apply: NOT_APPLICABLE (exclude from calculation)
- Raw score = (sum of weighted passes / total possible weight) × 100
- Apply confidence modifiers: +10 if lab contact confirms, +5 if QR verified,
  -20 if community testing contradicts claims
- Final score bands:
  90–100 = VERIFIED (accept)
  75–89  = ACCEPTABLE (accept with notation)
  55–74  = REVIEW_REQUIRED (request supplementary docs)
  30–54  = SUSPICIOUS (reject unless independently retested)
  0–29   = REJECT (do not use; report to community)
